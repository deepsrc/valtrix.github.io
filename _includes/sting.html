"STING, the flagship product of Valtrix Systems, is a bare-metal system validation tool for complex client/server SoCs."

Sting has been developed with a vision to solve problems seen in system validation. It embodies the best methodologies and practices in the industry whilst providing innovative solutions for the challenges in your ecosystem.

Aiming to be the tool-of-choice for the validation of your mobile, client and server systems.

Validation tool that shifts-left

Same tool for pre-silicon, emulation, FPGA and post-silicon validation; Test content scales across different hardware/software configurations; Improved throughput for content development and debug; Save on cost, time and resources spent on redundant validation

Enable concurrency testing in SoC(s)

Same tool to stress all IPs in the SoC concurrently; Supports ARMv8/AArch64 and PCI express currently but plan to support other IPs in near future to ensure validation coverage for any SoC implementation;

Diverse test stimulus specification mechanisms

The tool supports different types of test stimulus specification mechanism; Supports bias based mechanism to generate traffic for random testing needs; Implements a framework to include snippets of ASM like low level programming language for focused tests; Supports a C++ based API to include test scenarios which require complex programming constructs

Large library of test stimulus

Ships a large library of test stimulus with the tool; Coverage for common test scenarios like message passing, cacheline sharing between IPs, pipeline hazards etc. provided in the library; The library of test stimulus ensures easy ramp into validation readiness

Extremely user friendly

Specially designed user interface to make complex test specification easy; Easily interpretable test report file and debug logs with verbose information which can be used for manual or automated test intent analysis and failure debug; Extended documentation, FAQs and support manuals;

Easily debug failures

Consistent and reliable failure reproduction; Failure reproduction shift-lefts making it easy to recreate a post-si failure in emulation or pre-si; Debug hooks and mechanisms implemented in tool which allow rapid failure debug and resolution; Mechanism to incrementally reduce the test content from failure point for quick triage;

Create tool extensions easily

Tool designed in a manner to let customers create extensions of tool easily for proprietary IPs; Test content management layer lets the native features to be randomized effectively with the new extensions developed; C++ based API makes it easy to develop standalone device drivers;

Ready for use and deployment

Sting is ready for use and deployment; Various licensing and support schemes available to fit different kinds of engagement models; Committed to address the validation challenges by working together
